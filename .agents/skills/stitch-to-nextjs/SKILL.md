# Stitch MCP to Next.js Workflow

This guide outlines the streamlined, highly-efficient process for using the Google Stitch MCP to generate web designs and successfully implement them into a Next.js application. 

Because Stitch operates with limited context and can produce inconsistent code, this workflow heavily emphasizes the "After Checks"—the critical refinement phase where you enforce design consistency, mobile responsiveness, and accurate content.

## Phase 1: Generation & Validation

1. **Prompting Stitch**
   - Provide clear, contextual information about the project (brand identity, colors, typography).
   - **Content Injection (Crucial):** Stitch lacks full project understanding and will hallucinate false information if left to guess. 
     - *Recreating an existing site:* Provide the exact content from the original page and instruct Stitch to use it verbatim.
     - *Creating a new site:* Write out all the necessary copy, headlines, and data points first, then feed that complete information to Stitch.
   - **Keep it simple:** Instruct Stitch not to overcomplicate the UI layout natively. Clean, standard layouts port easier.
   - **Image prompting:** Explicitly request *photorealistic images* rather than abstract or "AI-looking" stock photos for placeholders.
2. **Retrieve & Validate**
   - Access the generated Stitch URL.
   - **Important:** There is no need to screen record or take photos/screenshots of the site.
   - Simply provide the generated Stitch link to the user so they can open it and review it themselves for immediate feedback. **Do not write code until the design is approved.**

## Phase 2: Efficient Implementation

1. **Extract Design Tokens First**
   - Before porting components, extract the core aesthetic from the approved Stitch page (background colors, primary/accent colors, font families, base border radii).
   - Implement these as CSS variables in `globals.css` to create a stable foundation.
2. **Download and Port**
   - Download the raw HTML/CSS from Stitch to use as a local reference.
   - Break the HTML structure into Next.js React components (e.g., `HeroSection`, `FeaturesSection`, `Header`, `Footer`).
   - Translate inline styles or utility classes into your project's styling solution (e.g., Vanilla CSS Modules).

## Phase 3: The "After Checks" (Refinement & Verification)

Stitch focuses on visual surface area, not production-readiness. You **must** perform these post-implementation checks:

1. **Content & Accuracy Verification**
   - **Text:** Stitch hallucinates placeholder text. Replace all copy with coherent, accurate, and relevant information.
   - **Links & CTAs:** Ensure all buttons and navigation links route to the correct pages using Next.js `<Link>`.
   - **Footer/Header details:** Update copyright years, legal links, and brand names to reflect the actual product.
2. **Visual Consistency & Stabilization**
   - **Icons & Logos:** Stitch uses text placeholders for icons (like "check_circle" or "diamond"). Replace these with actual SVGs or ensure the Material Symbols font is loaded in `layout.tsx`. Apply the real brand logo.
   - **Alignment:** Look out for broken flex/grid layouts. If elements overlap or fail to align evenly, manually rewrite the layout CSS to be structurally sound.
   - **Design Synchronization:** Stitch may generate inconsistent styles across different pages. It is your job to **stabilize** the design by forcing new pages to use the global design tokens established in Phase 2.
4. **Mobile Responsiveness**
   - Stitch designs are often desktop-only. 
   - Write media queries (`@media (max-width: 768px)`) for every component.
   - Implement hamburger menus for headers.
   - Force CSS Grids into single-column (`grid-template-columns: 1fr`) on mobile.
   - Adjust font sizes (`clamp()` or typography scaling) and padding for smaller viewports.
5. **Subtle Animations & Polish**
   - Stitch designs can feel static. Add "just enough" animation to impress without overwhelming the user or hurting performance.
   - Implement simple hover states (e.g., card lifts `transform: translateY(-4px)`, interactive glows, or image transitions).
   - Consider gentle fade-in animations on scroll for major sections to make the interface feel premium and alive.
   - **Rule of thumb:** Keep interactions snappy and subtle. Avoid long, complex, or janky animations that distract from the core product story.

## Handling Divergent Cases & Failures

### Case 1: Stitch Generates Inconsistent Aesthetics Across Multiple Pages
**The Problem:** The homepage uses one shade of dark mode, but the pricing page generates with a completely different background color and border radius.
**The Solution:** Discard the specific CSS variables generated by the divergent page. Map its HTML structure to the centralized CSS custom properties (`var(--surface-container)`, `var(--primary)`) you established in `globals.css` during the first successful generation.

### Case 2: Broken Layouts or Overlapping Content
**The Problem:** Stitch uses absolute positioning or complex margins that break when real text is inserted.
**The Solution:** Strip out the problematic positioning CSS entirely. Replace it with modern, robust layout primitives—use `display: grid` with `gap` for alignment, and `display: flex` with `justify-content: space-between` and `height: 100%` for equal-height cards.

### Case 3: Missing or Broken Icons Rendering as Text
**The Problem:** The UI shows the literal word "home" instead of a house icon.
**The Solution:** Ensure `<link href="https://fonts.googleapis.com/css2?family=Material+Symbols+Outlined..." rel="stylesheet">` is injected into the root HTML `<head>`.

### Case 4: AI Images Look Unprofessional
**The Problem:** Stitch generated cartoonish or confusing abstract images that distract from a premium UI.
**The Solution:** Use the `generate_image` tool independently to create photorealistic replacements, or use a reliable external service. Substitute the `src` attribute in the Next.js component.
